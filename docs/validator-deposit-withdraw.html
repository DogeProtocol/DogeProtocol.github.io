Validating staking, deposit and withdrawal

Important, please read before proceeding
  1. This is blockchain software. Do not use on a computer or device with sensitive or important personal data. Using this software will require opening firewall ports. Using this software may result in heavy internet data usage. Use at your own risk.
  2. After depositing, if you want to withdraw the coins, you will have to wait for 256000 blocks from the block from which the withdrawal was initiatied. This can be upto 60 days or more. This is for security reasons.
  3. Block Rewards are enabled only from a block that roughly co-incides with April 14th 2024, World Quantum Day. Unless you run the validator node, you will not get block rewards.
  4. You will have to update the latest node client software around the first week of April 2024, to take in bug fixes, new features and updates. After that, you will have to periodically update the node client software, when new features or updates or bug fixes are made.
  5. After depositing coins, if you do not run the validator node, you may loose coins to slashing and might not be able to recover your original deposit amount. The slashing is enabled from a block that roughly co-incides with April 14th 2024, World Quantum Day.
  6. If you had already registered as a genesis validator, these steps are not applicable. You may start validating by following the instructions at Connecting to the mainnet as a validator.
Common steps for depositing and withdrawal
  1. Pre-requisite; your computer should be connected to the mainnet and blocks should be getting created. Instructions at: Connecting To Mainnet.

  2. Open the appropriate shell window. Do not close the other shell window in which the mainnet is connected to.

    Windows
    Open Command Prompt
    Mac
    Open Terminal
    Linux
    Open Terminal

  3. Change to the folder where the blockchain node client is located.

    Windows 
    cd c:\dp
    Mac 
    cd $HOME/dp
    Linux 
    cd $HOME/dp

  4. This step is specific to Mac and Linux. Set the environment variable to the required library path. Remember that if you open a new Terminal window anytime, you will need to run this command again, to set the environment variable. You might want to add this environment variable to your shell profile.

    Mac 
    export DYLD_LIBRARY_PATH=$HOME/dp
    Linux 
    export LD_LIBRARY_PATH=$HOME/dp

  5. Set the environment variables to point to the data folder and DP_RAW_URL.

    Windows 
    set DP_KEY_FILE_DIR=$HOME/dp/data/keystore
    set DP_RAW_URL=//./pipe/geth.ipc
    Mac 
    export DP_KEY_FILE_DIR=$HOME/dp/data/keystore
    export DP_RAW_URL=~/.ethereum/geth.ipc
    Linux 
    export DP_KEY_FILE_DIR=$HOME/dp/keystore
    export DP_RAW_URL=~/.ethereum/geth.ipc

  6. Set the environment variables to point to the data folder and DP_RAW_URL.

    Windows 
    set DP_KEY_FILE_DIR=$HOME/dp/data/keystore
    set DP_RAW_URL=//./pipe/geth.ipc
    Mac 
    export DP_KEY_FILE_DIR=$HOME/dp/data/keystore
    export DP_RAW_URL=~/.ethereum/geth.ipc
    Linux 
    export DP_KEY_FILE_DIR=$HOME/dp/keystore
    export DP_RAW_URL=~/.ethereum/geth.ipc
Depositing
  1. Pre-requisite; you should already have a quantum wallet with coins for staking. A minimum of 5,000,000 (5 million) coins are required for staking.

  2. Create a quantum wallet for validating. This wallet will henceforth be called as the validator wallet. This wallet is different from the wallet from which you will be depositing the coins for staking. You will be prompted to enter a password. Ensure you use a strong password atleast 16 characters in length. You should never forget your password. If you don't remember it, the wallet will be useless and you cannot access any coins or tokens inside it.

    Windows 
    dp --datadir data account new
    Mac 
    ./dp --datadir data account new
    Linux 
    ./dp --datadir data account new
  3. Once you execute the above command successfully, the quantum wallet will be created. Note down the address of this wallet from the output of the above command. This will be your validator quantum address.
  4. You will also notice that a file has been created under the data\keystore location. This is your quantum wallet. An example is shown below: 
    Example: Path of the secret key file: data\keystore\UTC--2024-01-13T21-09-56.375263700Z--a553b8935988e6260b6e3c3ff5b340ee478b504b7166b4881365a9153c80a78c
  5. Backup the above wallet file in 3 different offline devices as explained in Prepare your device. You should not proceed to remaining steps until you backup the wallet.
  6. Remember that you will always need the password to use the wallet. If you don't remember it, the wallet will be useless and you cannot access any coins or tokens inside it. Consider noting down this password safely offline in a trusted location.
  7. Likewise, even if you remember the password, if the wallet file is lost or corrupted, you cannot access any coins or tokens. If the wallet is stolen and the password is not strong, the attacker can get all the coins or tokens in your wallet. Hence you should keep your wallet well protected.
  8. Copy the quantum wallet from which you are depositing the coins from, into the data\keystore folder.

  9. Run the following command to deposit coins.
    Replace DEPOSITOR_ADDRESS with the address of the wallet that has the coins to deposit. Ensure this wallet also has additional coins for gas fees for the transaction.
    Replace VALIDATOR_ADDRESS with the address of the wallet for validating as specified in step 2.
    Replace DEPOSITOR_AMOUNT with the number of coins you want to deposit.

    Windows 
    dputil stakingdeposit DEPOSITOR_ADDRESS VALIDATOR_ADDRESS DEPOSITOR_AMOUNT
    Mac 
    ./dputil stakingdeposit DEPOSITOR_ADDRESS VALIDATOR_ADDRESS DEPOSITOR_AMOUNT
    Linux 
    ./dputil stakingdeposit DEPOSITOR_ADDRESS VALIDATOR_ADDRESS DEPOSITOR_AMOUNT
  10. If the above command is successful, you should see a confirmation message. After about 10 minutes, you may check the despoit balance by following the steps in the section below: Checking Staking Balance.
  11. You should proceed to immediately run the validator node by following the steps in . Otherwise you might loose coins due to inactivity slashing.
  12. For security reasons, you should move the depositor quantum wallet to a different device and also keep multiple backups offline. You shouldn't keep the depositor quantum wallet in the same computer that is running the blockchain node. Only the validator wallet should be kept on this computer. If you loose your depositor quantum wallet or forgot the password or it gets stolen, even if you have the validator wallet, you will not be able to get your deposited coins back. Keep both your depositor and validator wallets safe!
Checking Deposited Staking Balance
  1. Pre-requisite; you should already have a quantum wallet with coins for staking. A minimum of 5,000,000 (5 million) coins are required for staking.

  2. You can initiate withdrawal by running the following command:

    Windows 
    dputil stakingbalance DEPOSITOR_ADDRESS
    Mac 
    ./dputil stakingbalance DEPOSITOR_ADDRESS
    Linux 
    ./dputil stakingbalance DEPOSITOR_ADDRESS
Withdrawing
  1. Pre-requisite; you should already have deposited coins for staking.
  2. Copy the depositor quantum wallet under the data\keystore folder if not already copied.

  3. You can initiate the withdrawal of coins by running the following command.

    Windows 
    dputil initiatestakingwithdrawal DEPOSITOR_ADDRESS
    Mac 
    ./dputil initiatestakingwithdrawal DEPOSITOR_ADDRESS
    Linux 
    ./dputil initiatestakingwithdrawal DEPOSITOR_ADDRESS

  4. If the above command is successful and the deposit completes, you can withdraw the staked coins after 256000 blocks have been created, by running the following command. This can be upto 60 days or more.

    Windows 
    dputil completestakingwithdrawal DEPOSITOR_ADDRESS
    Mac 
    ./dputil completestakingwithdrawal DEPOSITOR_ADDRESS
    Linux 
    ./dputil completestakingwithdrawal DEPOSITOR_ADDRESS
